# Media Note 写作风格规范（Obsidian + Media Extend）

目标：把 B 站课程字幕整理成**可复习的学习笔记**，同时保证“忠实、不遗漏、公式准确、可读性强”。  
定位：像“视频作者为自己的视频写配套讲义”，用“我们”讲清楚知识点与推导链条；不要写“作者想表达什么”式的第三人称转述。

## 输出必须包含的结构（强约束）

1. **Frontmatter 完整**（YAML，字段至少包含：`title/author/source/created/subtitle/tags/aliases/media`）。
2. `# <标题>` 后**紧接**：
   - `> [!summary]`（3-6 条要点），并在 summary 中列出**1-3 条核心公式（LaTeX）**，每条公式附**一句解释**（公式表达什么、关键符号代表什么）。
   - Mermaid 大纲图（建议 `flowchart TD`），节点写清 `HH:MM:SS + 主题`，并尽量与后续 `##` 标题一一对应。
3. **每个二级标题 `##` 的第一行**必须是 Media Extend 时间戳链接：
   - 固定格式：`[HH:MM:SS](<b站链接>)`
   - 链接必须使用**基础链接**（含 `p=XX`，但不带 `?t=`/`#t=`/`&t=`）。
4. 结尾必须有：`## 小结与预告`，最后一行追加：
   - `下一节：[[<下一个视频笔记>]]`
   - 若为课程最后一个视频，链接回课程总索引（例如 `[[强化学习的数学原理]]`）。

## 内容取舍与“忠实”标准（强约束）

- **删除**：寒暄/点赞关注/与知识无关的互动话术。
- **保留**：动机、定义、直觉解释、例子、推导步骤、结论、注意事项与易错点。
- **不遗漏**：除寒暄外，字幕里的每一句有效信息，都必须在笔记中有对应表述（允许书面化与合并短句，但不能丢逻辑与信息）。
- **不脑补**：不添加字幕未覆盖的新结论；只做口语转书面、补充必要的符号解释与排版。

## 排版与层级（可读性约束）

- `##` 用于“课程逻辑大段”；同一大段内的子问题/例子/推导步骤用 `###`，避免标题层级膨胀与混乱。
- 并列用 bullet list；步骤用有序列表；因果关系写清“因为/所以/因此”；对比用 `A vs B`。
- 关键词适度加粗；易错点/提醒用 callout（`[!note] / [!warning] / [!tip]`）。

## 公式规范（强约束，避免渲染失败）

- 数学命令前只写 **1 个反斜杠**：写 `\mathbb{E}`，不要写 `\\mathbb{E}`（`\\` 在 LaTeX 表示换行）。
- 不要写 `\*`（会渲染成红色反斜杠）。上标最优解/最优值写 `w^*` 或 `w^{*}`；若要“星号算子”，用 `\ast` 或 `\star`。
- 对数符号以 PPT/板书为准：讲的是 `\ln` 就写 `\ln`，不要擅自换成 `\log`。
- 对关键公式尽量使用 `\underbrace{...}_{...}` 标注项的含义，帮助读者对齐结构（main/target/基线/优势等）。
- **图不代文**：截图中的关键公式/表格/伪代码必须在正文用 LaTeX/Markdown/代码块复写出来，读者不看图也能理解。

## 图文对齐方法（解决“图代文 / 公式对不上 / 提到推导却没图”）

把每一张 PPT 截图当成一个“锚点”，写笔记时按下面的顺序做“图文对齐”：

1. **先锚定再解释**：在讲述该部分内容之前或最迟在同一小节内，先插入对应截图（带时间戳链接）。不要把截图丢到段落末尾当“装饰”。
2. **先写原式再化简**：如果视频在“化简/推导/变形”某个公式，笔记必须先把“要化简的原式”完整写出来，再围绕它逐步化简；不要直接从化简后的式子开始讲。
3. **推导用等式链对齐 PPT**：凡是正文出现“推导/证明/化简/展开/视频里的推导步骤”等表述，必须：
   - 插入对应的推导截图锚点（同小节内，且离推导正文不超过 5-10 行）。
   - 用 `aligned/align` 写出**逐行等式链**，结构与 PPT 一致；不能只写几条 bullet 概述。
4. **图中内容必须落到正文**：对每个锚点截图，至少满足其一：
   - 公式：在正文复写出截图里会被后文引用/讨论的核心公式（含上下标、求和范围、max/min、转置等）。
   - 表格/伪代码：在正文复写成 Markdown 表格或代码块。
   - 图示/流程：在正文用文字+列表把图要表达的关系写清楚（必要时用 Mermaid）。
5. **最后做一次“对照校验”**：写完该小节后，快速检查：
   - 截图里出现的所有关键符号/公式，在正文里是否都能找到对应 LaTeX（而不是只存在于截图）。
   - 正文符号与截图是否一致（例如 `\ln/\log`、`q/Q`、`\rho^\mu/\rho^\beta`、求和范围、是否有转置等）。

推导段落推荐模板（可直接复用）：

```markdown
- ![[...webp|<caption>|650]] [HH:MM:SS](https://www.bilibili.com/video/BV.../?p=XX)

下面把 PPT 上这条等式按定义展开并逐行化简：

$$
\begin{aligned}
  \cdots
\end{aligned}
$$

其中关键点：
- ...
- ...
```

## 截图规范（强约束）

- 首次版本必须插入关键截图（建议每节 4-12 张）：关键公式、关键图表、伪代码、例子表格、需要据图校准的推导。
- 每一张截图的 embed 附近必须有**紧邻**时间戳链接（同样不带 `?t=`/`#t=`/`&t=`）。
- 关键锚点截图必须是**高清**（优先 `bili_snapshot.py` 的 `video` 方法导出的 1920x1080 `.webp`）。  
  低清 `preview 480x270` 只能临时占位，不得作为关键公式/图表的最终锚点。
- 插图后要回头“以图为准”校准该小节的所有相关公式与符号（上下标、转置、求和范围、max/min 等）。

## 链接与别名（Obsidian 约束）

- Media Note 的 `aliases` 只写“本视频笔记标题的变体”（去掉 `Media Note -`、标点/空格差异等），不要写通用概念名词。
- 对课程中反复出现的核心概念，建立“概念卡片”笔记（独立 `.md`）：
  - 使用 `> [!definition] / [!theorem] / [!proposition] / [!remark]` 等 callout 写清：定义（公式）、性质、常见等价变形与使用场景。
  - 在 Media Note 正文首次出现处用 `[[概念名]]` 建立双链。

## 最终自检（提交前必须通过）

- 所有 `$...$` 成对；所有 `$$`/代码围栏/mermaid 围栏闭合。
- 每个 `##` 下一行是时间戳链接。
- 笔记中不出现 `?t=` / `#t=` / `&t=` 的链接参数。
- 数学环境中不出现 `\\mathbb`、`\\nabla` 这类错误写法，也不出现 `\*`。
